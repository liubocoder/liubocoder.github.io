Kotlin基础

如果有Java基础，Kotlin语法上基本没啥难度，主要是一些特殊的写法、关键字需要熟悉，类、集合怎么定义和使用。

创建测试项目

这里不再描述AndroidStudio中如何创建了，仅记录下IDEA中创建，IDEA中推荐使用gradle的方式创建Kotlin/JVM项目用于测试Kotlin，若不选用项目构建工具，需要自己管理依赖，太麻烦了。。比如协程稍微高级一点的api都不在标准库中， 协程相关依赖implementation("org.jetbrains.kotlinx:kotlinx-coroutines-core:1.3.61")，版本根据实际指定

类似Java静态成员 companion object 关键字

companion object 用于定义类的静态属性和方法 单独的object用于定义静态对象的成员属性和方法

fun main() {
    println("start")
    println("MyCls name=${MyCls.name}, lastname=${MyCls.lastname()}")
    println("MyCls.sObj name=${MyCls.sObj.name}, lastname=${MyCls.sObj.lastname()}")
}

class MyCls {
    object sObj {
        val name = "obj.single_obj"
        fun lastname(): String {
            return name.split(".")[1]
        }
    }
    companion object {
        val name = "obj.companion_obj"
        fun lastname(): String {
            return name.split(".")[1]
        }
    }
}

委托 by

委托在kotlin中非常有用，可以用个委托完成语言级别的一些设计模式支持

  1. 类委托class Hsq(a: ActionEat): ActionEat by a
  2. 属性委托val/var <属性名>: <类型> by <表达式> …

延迟加载 by lazy(lambda)

简写by lazy {…}，仅用于常量val，当变量第一次调用的时候加载

@JvmOverloads

用于兼容java的重载机制，kotlin中不需要重载

内联 inline

用于修饰函数，使用的感觉有点像C的宏定义，kotlin将inline函数的定义直接拷贝到代码的调用之处， 因此要注意break、return这类跳转语句。

lateinit

不能修饰8种基本的数据类型，可以用?替换

Lambda表达式

Kotlin的Lambda表达式功能更强，可以将函数作为参数传递，可以修改作用域之外的参数

fun mylambda(param1: String, paramLambda: (String) -> Unit) {
    //定义一个接收Lambda表达式的函数
    ...
}
//这类最后一个参数是Lambda的函数，调用时可以简写为以下格式，it是参数的默认名
mylambda("this s key") {
    //t-> 修改参数默认的名称为t
    println(it)
}

let、with、run、apply、also函数

  1. let内联函数扩展 
    object?.let {
  //it替换object，最后一行的值返回出去
  it.sometings()
}
  2. with 
    with(object) {
  //这里相当于object类的内部一样，this为object
  //可直接使用object的一些公开的属性和方法
  //例如object有个name的属性
  println(name)
}
  3. run 相当于let和with的结合 
    object?.run {
 //this为object
}

  4. apply 与run类似，只是返回值不同，apply返回的是对象本身

  5. also 与let类似，只是返回值不同，also返回的是对象本身

协程

个人感觉Kotlin协程是对Java线程的功能封装，并在语法上对异步功能的加强。 协程3种启动方式

  1. CoroutineScope + launch{}/async{}/… 这种是最基础功能，通过实现CoroutineScope接口，将协程的生命周期与业务组件生命周期绑定，还有其他开启协程的方式都是基于此的
  2. GlobalScope.launch{} / GlobalScope.async() /..
  3. runBlocking{}

launch{}返回job对象 async{}返回Deffer对象，可调用await等待协程结束

异常处理

  1. 协程取消，suspend方法抛出CancellationException
  2. 常规异常
    • launch抛给父协程，导致父协程退出
    • async 可await()捕获 
      launch(CoroutineExceptionHandler{_, e->
 //异常处理
 ...
 }) {
 //开启协程
 ...
 }
